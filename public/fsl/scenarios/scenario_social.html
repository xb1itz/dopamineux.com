Scenarios:

    Social scenarios

    These scenarios based on social features of FSL, allowing fans to connect and keep track of each other.

    Legend

    Each scenario represents a particular business case. What steps users take to interact with the system to complete their goals.
    Some of the steps have additional indicators that are explained below:

    • A trigger

      Represents a point or event where users are being engaged into the scenario.

    • Investment

      A step where a user has to spend mental effort to proceed to further steps of the scenario. This is usually the most challenging moment in the scenario.

    • AFK timeout

      Away From Keyword moment that represents a temporary break in the scenario, where users are not actively engaged in a system.

    • Notification

      Represents an automatic notification that engages user outside of a system.

    • The reward

      A motivator that rewards user for completing whole (or a part of) scenario.

    #1 Public user's feed

    • 1. Entry point

      The scenario begins when a user clicks on username of any user in the system.

    • 2. User's feed

      System displays all the information related to user:

      • The content user have posted (User's feed)
      • CTA's to follow/unfollow the particular user
      • User's team
      • Personl data (real name, avatar, tag line)

    #2 Following a user

    • 1. Entry point

      The scenario begins when user has an intention to follow a particular user

    • 2. Finding a user

      User can be found in different ways:

      • Search feature in navigation
      • Context menu in post feed
      • Visiting user's profile directly

    • 3. Focusing on user

      User click's through to reach a particular user context (e.g. User profile)

    • 4. Follow

      User clicks on follow CTA

    • 5. Reward message

      User click on follow CTA

      5.1. New follower notification

      Notification is sent to the user who was followed

    #3 Unfollowing a user

    • 1. Entry point

      The scenario begins when user has an intention to UNFOLLOW a user, because something negative cathces their eye.

    • 2. Focusing on user

      To unfollow someone user has to select user in one of the following ways:

      • Search feature in navigation
      • Context menu in post feed
      • Visiting user's profile directly

    • 3. Unfollow

      User clicks on unfollow CTA

    • 4. Success message

      A success response is showed, indicating that operation is complete.

    #4 Send message

    • 1. Entry point

      The scenario begins when user has an intention to send a message to another user

    • 2. Finding a user

      User can be found the same ways decribed in Scenario #1

    • 3. Clicking through

      User clicks on send message CTA

    • 4. Chat form

      System provides single field message form showing chronological chat of two users.

    • 5. Entering a message

      User enters his message into form and submits it.

    • 5. Reward message

      Reward message is showed.

      5.1. New message notification

      A new message notification is sent to recipient user

    #5 Receive a message

    • 1. New message notification

      The scenario begins user receives a notification about new message

    • 2. Click through

      User clicks to see the message

    • 3. Inbox

      System opens up a chat screen where user see full conversation with sender.

    • 4. Reply

      User can instantly reply a message within the same screen. Same as in Scenario #4

    #6 Read all messages

    • 1. Entry point

      The scenario begins when user has intention to check his entire inbox

    • 2. Click through

      User clicks to see his messages in navigation

    • 3. Inbox

      System opens up a list of past conversations with other users.

    • 4. Chat

      User can select a particular conversation and continue his convestation as in Scenario #4 and Scenario #5.

    #7 Check all news messages

    • 1. Entry point

      The scenario begins when user multiple unred messages

    • 2. Click through

      User clicks to see his messages in navigation

    • 3. Inbox

      System opens up a list of all conversation with unread conversations highlighted.

    #8 Archive message

    • 1. Entry point

      The scenario begins when user wand to remove conversation with particular person from their history.

    • 2. Click through

      User clicks to inbox to get into particular conversation context.

    • 3. Inbox

      System opens up a list of all conversation with unread conversations highlighted.

    • 5. Archive action

      User select an alternate action Archive to hide conversation from their inbox.

    • 6. Success message

      A success message is shown in the inbox, instead of archived conversation.
