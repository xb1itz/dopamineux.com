Scenarios:

    Advanced PRE Match scenarios

    Scenarios targeting users before particular match starts with additional gamification.
    Note that all these scenarios are subject for experiments due to need for balancing of gamification factors.

    Legend

    Each scenario represents a particular business case. What steps users take to interact with the system to complete their goals.
    Some of the steps have additional indicators that are explained below:

    • A trigger

      Represents a point or event where users are being engaged into the scenario.

    • Investment

      A step where a user has to spend mental effort to proceed to further steps of the scenario. This is usually the most challenging moment in the scenario.

    • AFK timeout

      Away From Keyword moment that represents a temporary break in the scenario, where users are not actively engaged in a system.

    • Notification

      Represents an automatic notification that engages user outside of a system.

    • The reward

      A motivator that rewards user for completing whole (or a part of) scenario.

    #1 Create a challenge

    • 1. Entry point

      User just made a pfediction

    • 2. Challenge CTA

      System displays a CTA for challenging a friend

    • 3. Click through

      User click's through to open challenge screen

    • 4. Challenge screen

      System displays an option to invite friends to make same prediction as user just did.

      • Select friend from follower list
      • Enter username
      • Enter email

    • 5. Send invitation

      User select a list of friends who they weant to invite and sends invitations.

    • 5. Success message

      A success message is displayed.

      5.1. Notification

      A notification is send to each invited friend registeded in the system.

      5.2. Invitation email

      If user with specified email is not registered, invitation is sent via email.

    #2 Share challenge

    • 1. Entry point

      User just made a pfediction

    • 2. Share a challenge CTA

      System displays a CTA to share a prediction on social networks and challenge friends

    • 3. Click through

      User click's through to share screen, same as sharing other platform content

    #3 Inviting friends challenge

    • 1. Entry point

      User made a pfediction

    • 2. Navigating to prediction

      User navigates to prediction they previuosly made

    • 3. Invite CTA

      System displays a CTA to challenge the friends
      From thoi point scenario continues as in REward step of Sceantio #1

    #4 Starting XI

    • 1. Entry point

      The scenario begins when user has found a particular match.

    • 2. Make Starting XI CTA

      System offers to make Starting XI

    • 3. Clicking through

      User clicks on CTA that leads to Starting XI screen

    • 4. Starting XI screen

      System navigates user to screen where they can select players from particular team starting a match.
      The selection here happens in UI designed to set and swith players with minimum cound of steps.

    • 5. Reward message

      Reward message is showed.
      Additionally sysem shows any Starting XI made by followed users.
