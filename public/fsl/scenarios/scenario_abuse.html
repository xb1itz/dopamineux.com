Scenarios:

    Abuse Moderation scenarios

    Since many of respondents complained that other fans can be toxic or abusive, we need to implement mechanisms to remove such users.

    Legend

    Each scenario represents a particular business case. What steps users take to interact with the system to complete their goals.
    Some of the steps have additional indicators that are explained below:

    • A trigger

      Represents a point or event where users are being engaged into the scenario.

    • Investment

      A step where a user has to spend mental effort to proceed to further steps of the scenario. This is usually the most challenging moment in the scenario.

    • AFK timeout

      Away From Keyword moment that represents a temporary break in the scenario, where users are not actively engaged in a system.

    • Notification

      Represents an automatic notification that engages user outside of a system.

    • The reward

      A motivator that rewards user for completing whole (or a part of) scenario.

    #1 Report another user

    • 1. Entry point

      The scenario begins when user has was abused by another FSL user

    • 2. Finding a user

      User can be found in different ways:

      • Search feature in navigation
      • Context menu in post feed
      • Visiting user's profile directly

    • 3. Focusing on user

      User click's through to reach a particular user context (e.g. User profile)

    • 4. Report

      User clicks on Report CTA that opens a report form

    • 5. Report form

      System provides a free-text form where user can descibe the case of abuse and submits that to system admins.

    • 5. Reward message

      Success message is displayed

      5.1. System administrators are informed about abuse

      Notification is sent to system administrators with attached details for both users and abuse description.

    #2 Report content

    • 1. Entry point

      The scenario begins after user feels abused by a certain piece of content.
      This scenario is identical to Scenario #1, just oriented into reporting a specific piece of content.

    #3 Report comment/reactiont

    • 1. Entry point

      The scenario begins after user feels abused by a certain piece of comment/reaction.
      This scenario is identical to Scenario #1, just oriented into reporting a specific piece of comment/reaction.
