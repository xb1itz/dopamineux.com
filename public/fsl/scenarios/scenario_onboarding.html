Scenarios:

    Onboarding scenarios

    These are scenarios designed to hook a new FSL user by providing them perceptive value as soon as possible. The challenge here is to present FSL as new platform that will be used by every football fan.

    Legend

    Each scenario represents a particular business case. What steps users take to interact with the system to complete their goals.
    Some of the steps have additional indicators that are explained below:

    • A trigger

      Represents a point or event where users are being engaged into the scenario.

    • Investment

      A step where a user has to spend mental effort to proceed to further steps of the scenario. This is usually the most challenging moment in the scenario.

    • AFK timeout

      Away From Keyword moment that represents a temporary break in the scenario, where users are not actively engaged in a system.

    • Notification

      Represents an automatic notification that engages user outside of a system.

    • The reward

      A motivator that rewards user for completing whole (or a part of) scenario.

    #1 Organic visitor to landing page

    • 1. User hits landing page

      The scenario begins when a user visits FSL home page

    • 2. Teaser

      Here we show a teaser of FSL features, stats link to App store / Play store

    • 3. User clicks through

      User clicks through to particular store item.

    • 4. Install

      User enters Scenario #2

    #2 Download funnel from Store

    • 1. Store

      The scenario begins when user lands in App/Play store

    • 2. Install

      Users go through store install process
      Initial permission required:

      • Internet
      • Profile
      • Notifications
      • Location
      • Camera (optional)

    • 3. App open

      User opans the app

    • 4. Reward sceeen

      App shows post-install welcome message

    • 5. Social auth

      Users go through social auth process.
      Note: we don't provide a sign-up form (At least in MVP)

    • 6. Welcome message

      We show nice welcome message that elevates user emotionally.

    • 7. Team selection

      We ask users to select the team they support.

    • 8. A team feed is shown

      After users slect their team, we instantly show the main feed fot a team.
      Feed includes:

      • Recent hot news related to team
      • Information on upcomming matches
      • Information on past matches
      • Chronologically displayed special team events (transfers, purchaces...)
      • Trigger to follow other,most engaging users of the team
      • Trigger to participate in gamification events (predictions, challenges, starting XI)

      8.1. A welcome letter is sent

      The system sends a welcome letter personally crafted content.

    #3 Virality trigger

    • 1. User opens FSL again after some time

      User opens FSL again after some time, after we collect enaough information about them

    • 2. Suggest to invite friends to join FSL

      Here we show an invitation screen, where user can send invitations to his friends.
      If user is signed-up via Facebook, we can ask to share a list of friends to send invitations directly to them.

    • 3. Reward and elevation

      We show nice reward message and give bonus reputation reward. (In case if we implement gamification here)

    #4 Invite from friend

    • 1. User receives an invitation

      User receives invitation link to FSL from their friend via social channels or email.

    • 2. User lands on invitation page

      Here we provide personalised message with CTA inviting to join FSL

    • 3. Social auth

      User gos through same social auth state as in Scenario #1

    • 4. Team selection

      Same state as in Scenario #1, just with highlighted team of user who sent the invite.

    • 5. Welcome reward

      After selecting team users see the welcome message that elevates them.
      The friend who invited them, automatically is added to contact/follower list.

      5.1. A welcome letter is sent

      The system sends a welcome letter personally crafted content.

      5.2. Notification for friend

      A friend who send invitation, gets notification that it was accepted.

    #5 Link from social media

    • 1. User discover FSL content on social media

      Scenario begins when user stumbles upon the FSL content piece that was shared on socual networks.

    • 2. User clicks through

      User clicks through to particular content page.

    • 3. System displays CTA

      System shows CTA with teasing copywrite that invites user to join FSL to discover more related content.

    • 4. User clicks through

      Scenario continues the same as Scenario #1

    #6 Invite to watch live score

    • 1. User receives an invitation

      User receives invitation link to FSL that leads to live game feed.

    • 2. User lands on live game feed

      User lands on game feed that shows all the information on the match:

      • Match score
      • March events
      • Other user reactions/comments
      • Information on who sent the invite
      • Highlighted contribution from friend who sent an invite

    • 3. Social auth

      If user tries to engage on something, they are forcer so sign-in to continue.
      User gos through same social auth state as in Scenario #1 only in modal way.

    • 4. Engagement

      After the auth, user is dropped to the same place, where they can continue on the action they begin at.

      5.1. A welcome letter is sent

      The system sends a welcome letter personally crafted content.

      5.2. Notification for friend

      A friend who send invitation, gets notification that it was accepted.
      If that friend is watching a live match feed, they see accepted invitation in the feed.

    #7 SEO based acquisition

    • 1. User discover FSL content on search engines

      Scenario begins when user stumbles upon the FSL content piece that was indexed by search engines.

    • 4. User clicks through

      Scenario continues the same as Scenario #4

    #8 GAME based acquisition

    • 1. User receives an invitation to the game/challenge

      User receives invitation link to FSL from their friend in the context of FSL mini-game:

      • Predictor challenge
      • Starting XI
      • Direct invitation to play specific mini-game

    • 2. User clicks through

      User clicks through to initial page of the particular game.

    • 3. System displays CTA

      Depending of the game, FSL displays a tailer message that invites users to join the specific game and complete with their friend.

    #9 Organic visitor with VP

    • 1. User hits landing page

      The scenario begins when a user visits FSL home page

    • 2. The feed

      System displays top hot content on football league that provides initial value

    • 3. User is engaged with social reaction

      Users can see reactions and comments that engages them.

    • 4. User try to engage

      When users tries to do a registered tier action, they are asked to sign-up and ends up in the same Social auth state in Scenario #1
