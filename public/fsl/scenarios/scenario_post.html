Scenarios:

    POST Match

    Scenarios targeting users after particular match ends.

    Legend

    Each scenario represents a particular business case. What steps users take to interact with the system to complete their goals.
    Some of the steps have additional indicators that are explained below:

    • A trigger

      Represents a point or event where users are being engaged into the scenario.

    • Investment

      A step where a user has to spend mental effort to proceed to further steps of the scenario. This is usually the most challenging moment in the scenario.

    • AFK timeout

      Away From Keyword moment that represents a temporary break in the scenario, where users are not actively engaged in a system.

    • Notification

      Represents an automatic notification that engages user outside of a system.

    • The reward

      A motivator that rewards user for completing whole (or a part of) scenario.

    #1 Check score

    • 1. Entry point

      The scenario begins when user has an intention to check any information on particular match that has ended.

    • 2. Finding a match

      Match can be found in different ways:

      • Search feature in navigation
      • User's team feed
      • Match list

    • 3. Focusing on particular match

      User click's through to reach a particular match context

    • 4. Match details

      All match details are displayed.

      • Match time, venue
      • Information about playing teams
      • Discussions
      • Frien's predictions
      • CTA to make starting layout
      • CTA to make a prediction

    #2 Check reactions

    • 1. Entry point

      The scenario begins after particular match ends.

    • 2. Finding a match

      User navigates to match same way as in Scenario #1

    • 3. Match feed

      All user reactions of match are displayed separate UI element

    #3 Key moments

    • 1. Entry point

      The scenario begins after particular match ends.

    • 2. Finding a match

      User navigates to match same way as in Scenario #1

    • 3. Match feed

      All top rated moments are highlighted and displayed on match screen.

    #4 Reaction

    • 1. Entry point

      The scenario begins after particular match ends.

    • 2. Finding a match

      User navigates to match same way as in Scenario #1

    • 3. Reaction form

      System provides single field message form.

    • 4. Entering a message

      User enters his message into form and submits it.

    • 5. Reward message

      Reaction/message appears on users feed with link to particular match.

    #5 Rate

    • 1. Entry point

      The scenario begins after particular match ends.

    • 2. Finding a match

      User navigates to match same way as in Scenario #1

    • 3. Rate form

      System provides custom form where user can rate each individual player in the context of the match.
      User also may add comments for each player describint their performance.

    • 4. Submiting a form

      User rates each or few players and submits it.

    • 5. Reward message

      Rating appears on users feed with link to particular match
      Rating also displayed on the match screen itself.

    #6 Share a score

    • 1. Entry point

      The scenario begins after particular match ends.

    • 2. Finding a match

      User navigates to match same way as in Scenario #1

    • 2. Share a match CTA

      System displays a CTA to share a match on social networks

    • 3. Click through

      User click's through to share screen, same as sharing other platform content

    #7 Sharing reaction

    • 1. Entry point

      The scenario begins after particular match ends.

    • 2. Finding a match

      User navigates to match same way as in Scenario #1

    • 2. Share a match CTA

      User use same CTA to share his reaction about the match as in Scenario #7. They just enter different message
