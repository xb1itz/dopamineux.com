Scenarios:

    PRE Match scenarios

    Scenarios targeting users before particular match starts.

    Legend

    Each scenario represents a particular business case. What steps users take to interact with the system to complete their goals.
    Some of the steps have additional indicators that are explained below:

    • A trigger

      Represents a point or event where users are being engaged into the scenario.

    • Investment

      A step where a user has to spend mental effort to proceed to further steps of the scenario. This is usually the most challenging moment in the scenario.

    • AFK timeout

      Away From Keyword moment that represents a temporary break in the scenario, where users are not actively engaged in a system.

    • Notification

      Represents an automatic notification that engages user outside of a system.

    • The reward

      A motivator that rewards user for completing whole (or a part of) scenario.

    #1 Upcoming matches

    • 1. Entry point

      The scenario begins when a user navigates to see all match list.

    • 2. Match list

      System displays all the information related to all upcomming game in single list, where all upcomming matches are shown gighlighted

    #2 Match details

    • 1. Entry point

      The scenario begins when user has an intention to check any information on particular match

    • 2. Finding a match

      Match can be found in different ways:

      • Search feature in navigation
      • User's team feed
      • Match list

    • 3. Focusing on particular match

      User click's through to reach a particular match context

    • 4. Match details

      All match details are displayed.

      • Match time, venue
      • Information about playing teams
      • Discussions
      • Frien's predictions
      • CTA to make starting layout
      • CTA to make a prediction

    #3 Tickets

    • 1. Entry point

      The scenario begins when user has found a particular match.

    • 2. Ticket CTA

      System offers to purchase match tickets if they are available

    • 3. Clich through

      User clicks Ticket CTA that leads to external ticketing system.

    #4 Predictor

    • 1. Entry point

      The scenario begins when user has found a particular match.

    • 2. Make prediction CTA

      System offers to make prediction

    • 3. Clicking through

      User clicks on CTA that leads to prediction screen

    • 4. Prediction screen

      System navigates user to screen where they can enter prediction score for particular match.

    • 5. Entering a score

      User places his prediction into form and submits it.

    • 5. Reward message

      Reward message is showed.
      Additionally sysem shows any predictions made by followed users.

    #5 Friend's predictions

    • 1. Entry point

      The scenario begins when user has found a particular match.

    • 2. Friends predictions

      If people mustually followed by user made a prediction on particular match, their prediction are showed on match screen.
      If may predictions are made by friends, system provides additional link to see all predictions.

    #6 Other predictions

    • 1. Entry point

      The scenario begins when user has found a particular match.

    • 2. Other predictions

      Basically case as in Scenario #5, just unrelated people predictions are shoved in lower visual gravity.

    #7 Predictions and tactics

    • 1. Entry point

      The scenario begins when user has found a particular match. It's purpose is to drive more users to create discussion before match.
      In this case the scenario is the same as in Scenario #1, just use different form template for making a starting IX set.
