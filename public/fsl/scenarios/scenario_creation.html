Scenarios:

    Content creation

    Scenarios enabling users to generate FSL content based upon various intentions.

    Legend

    Each scenario represents a particular business case. What steps users take to interact with the system to complete their goals.
    Some of the steps have additional indicators that are explained below:

    • A trigger

      Represents a point or event where users are being engaged into the scenario.

    • Investment

      A step where a user has to spend mental effort to proceed to further steps of the scenario. This is usually the most challenging moment in the scenario.

    • AFK timeout

      Away From Keyword moment that represents a temporary break in the scenario, where users are not actively engaged in a system.

    • Notification

      Represents an automatic notification that engages user outside of a system.

    • The reward

      A motivator that rewards user for completing whole (or a part of) scenario.

    #1 Short post

    • 1. Entry point

      The scenario begins when a user opens the app/site with intension to post a public message.

    • 2. Click through

      User clicks to post something on their mind

    • 3. Posting form

      System provides single field post form

    • 4. Entering a post message

      User enters his message into form and submits it.
      User can paste a link that is processed in background crawler to fetch a OG preview.
      Additionally, during this step user can tag other users using @.

    • 5. Post published

      New post is showed highlighted in user's feed.
      Post appears on users follower's feed.

      5.1. Notification for mentioned users

      Users who were tagged in post receive a nitification

    #2 Capture a photo | App only

    • 1. Entry point

      The scenario begins when a user opens the app/site with intension to post a public photo.

    • 2. Click through

      User clicks to post something and selects camera option

    • 3. Camera snapshot

      System opens a camera screen that allows user to capture image directly.

    • 4. Shot

      User capture a photo

    • 5. Upload state

      System provides upload state for captured photo

    • 6. Confirm state

      System shown photo post, where user can see how their photo looks in feed after publishing it.

    • 7. Add a message | Optional

      Additionally user can enter a message or title of the photo in the same way they add a short post in Scenario #1

    • 8. Photo published

      New photo is showed highlighted in user's feed.
      Photo appears on users follower's feed.

      8.1. Notification for mentioned users

      Users who were tagged in post receive a nitification

    #3 Upload a photo

    • 1. Entry point

      The scenario begins when a user opens the app/site with intension to post a public photo.

    • 2. Click through

      User clicks to post something and selects upload photo option

    • 3. File selector

      Systm opens file selector, where user can select a particular photo to upload.

    • 4. Upload...

      From this point scenario continues as Scenario #2

    #4 Create a discussion

    • 1. Entry point

      The scenario begins when a user opens the app/site with intension to create a discussion thread.
      This scenario uses same steps as in Scenario #1, just with different type of message.

    #5 A long post

    • 1. Entry point

      The scenario begins when a user opens the app/site with intension to post a SERIOUS public message with long content.

    • 2. Click through

      User clicks to post something on their mind

    • 3. Swithhing posting mode

      User click to swithc editor tu rich text/markdown

    • 4. Entering a post message

      Here system provides a different editor to enter a content.
      This editor should allow at least basic rich text functions and/or support markdown

    • 5. Post published

      From this point scenario continues as Scenario #1

    #6 Delete

    • 1. Focus on the post

      Scenario begins when user views his own post (Only text based content).

    • 2. Click through

      User clicks to delete a post.

    • 3. Delete prompt

      System shows a delete prompt if user really wants to remove the particular post.

    • 4. Post deleted

      A success message is shown in the feed, instead of deleted post.

    #7 Edit

    • 1. Focus on the post

      Scenario begins when user views his own post (Only text based content).

    • 2. Click through

      User clicks to edit a post.

    • 3. Edit form

      System provides an edit form to modify a context

    • 4. Editing a text

      User emakes corrections to text content

    • 5. Save the changes

      New modified post is showed highlighted in the feed.
